Core Web Vitals

3 metrics Google actually uses for ranking

LCP Fix
  • → Preload hero image
  • → No lazy-load above fold
  • → Use CDN + WebP/AVIF
CLS Fix
  • → Add width/height to every image
  • → font-display: swap
  • → Reserve space before inject
INP Fix
  • → Avoid heavy JS
  • → requestIdleCallback
  • → Code splitting
LCP — Right vs WrongHTML
<!-- ✅ Correct: Preload hero image -->
<link rel="preload" as="image" href="/hero.webp" fetchpriority="high">

<!-- ❌ Wrong: lazy-load an above-the-fold image -->
<img src="/hero.jpg" loading="lazy">

<!-- ✅ Correct: specify dimensions to prevent CLS -->
<img src="photo.webp" width="800" height="600" alt="...">

<!-- ❌ Wrong: no dimensions — layout jumps when image loads -->
<img src="photo.jpg">

Meta Tags

The first thing Google sees — controls title, description and social preview

Essential Meta TagsHTML
<!-- Title: 50-60 characters, main keyword first -->
<title>Fresh Coffee AunJai | Sukhumvit Café Bangkok</title>

<!-- Description: 120-158 characters, compelling enough to click -->
<meta name="description" content="Freshly roasted coffee weekly, free WiFi, open 7:00-21:00, near BTS Asok">

<!-- Canonical: prevent duplicate content -->
<link rel="canonical" href="https://www.aunjai-cafe.com/menu">

<!-- Open Graph: preview on Facebook/Line -->
<meta property="og:title" content="AunJai Café">
<meta property="og:image" content="https://aunjai-cafe.com/og-image.jpg">
<meta property="og:url" content="https://aunjai-cafe.com">

<!-- Twitter Card -->
<meta name="twitter:card" content="summary_large_image">

Same concept — but code style differs by stack

10.1 Next.js — Static & Dynamic Metadata

Static MetadataTSX
// app/menu/page.tsx
import { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Coffee Menu | AunJai Café',
  description: 'Freshly roasted coffee weekly, starting at 65 THB',
  alternates: { canonical: 'https://aunjai-cafe.com/menu' },
}
Dynamic Metadata — fetched from APITSX
// app/menu/[slug]/page.tsx
export async function generateMetadata({ params }) {
  const item = await fetch(`https://api.aunjai.com/menu/${params.slug}`)
    .then(r => r.json())

  return {
    title: `${item.name} | AunJai Café`,
    description: item.description,
    openGraph: { images: [item.imageUrl] },
  }
}

10.2 SPA — Solutions when migration is not possible

Dynamic Rendering — nginx confignginx
# Detect User-Agent and route Bot to prerender service
map $http_user_agent $is_bot {
  default     0;
  ~Googlebot  1;
  ~bingbot    1;
}
server {
  location / {
    if ($is_bot) {
      proxy_pass http://prerender-service; # Pre-rendered HTML
    }
    try_files $uri /index.html; # Regular users get SPA
  }
}

Content SEO — The Pillar That Technical SEO Alone Can't Replace

Technical SEO makes a page readable to Google — but content that actually answers what people search for is what makes it rank

11.1 Keyword Research — Know Before You Write, Don't Guess

Before writing any article, you need to know what people actually search for — not what you assume they'd search for.

  • →Google Keyword Planner — Free, requires a Google Ads account (no need to run actual ads to view data)
  • →Google Search Console → Performance — see queries you already rank for but haven't fully optimized
  • →AnswerThePublic — see real questions people search around a single keyword
  • →Google Autocomplete — type a term and read the suggestions, the easiest free method there is

11.2 Search Intent — Write to Match What People Actually Want

IntentExample keywordWhat to write
Informational"how to brew espresso"Step-by-step how-to article
Navigational"AunJai Café"Clear, easy-to-find brand homepage
Commercial"best coffee capsule brand"Comparison/review article
Transactional"buy coffee capsules free shipping"Product page with clear buy button
⚠️ Common mistake: writing a sales-focused (transactional) page for a purely informational keyword — Google will see the content as a mismatch for what the searcher wanted, and won't push it up in rankings.

11.3 E-E-A-T — The Framework Google Uses to Assess Content Quality

E-E-A-T stands for Experience, Expertise, Authoritativeness, Trust — a framework from Google's Search Quality Rater Guidelines

💡 Important nuance: E-E-A-T is not a direct, measurable ranking factor with its own score. It's a conceptual framework used by both human raters and Google's algorithms to assess overall site quality — especially for YMYL content (Your Money or Your Life — health, finance, legal, safety, and as of a 2025 update, content related to elections/public institutions too), where Google applies its strictest standards.
Experience + Expertise
  • → Real photos from testing a product
  • → Author bio with real name + credentials
  • → Not attributed to a vague "Team"
Authoritativeness + Trust
  • → Backlinks from credible sites in the field
  • → HTTPS + clear contact info
  • → Cite verifiable sources/statistics
⚠️ AI-generated content: Google doesn't ban AI-assisted writing, but content that reads as purely AI-generated, with no human review and no original insight, tends to be classified as low quality. What Google cares about is whether the final result is accurate, trustworthy, and has someone accountable for it.

11.4 Good Content Structure

Good Article StructureOutline
h1: Main heading (matches search intent)
  Opening paragraph: answer the core question immediately, no preamble
  h2: Sub-topic 1
    Detailed content
  h2: Sub-topic 2
    Detailed content
  Summary / FAQ at the end
  • ✓Answer the core question within the first 1-2 sentences — Google's AI Overview wants a fast answer
  • ✓Use h2/h3 to split content by questions readers likely have, not by how you'd prefer to ramble
  • ✓Length isn't a quality signal — short and complete beats long and circular
  • ✓Updating stale content beats abandoning it

11.5 Internal Linking

Internal Link — Right vs WrongHTML
<!-- ✅ Good: descriptive anchor text -->
<a href="/menu/single-origin">Single Origin coffee from Ethiopia</a>

<!-- ❌ Bad: anchor text says nothing -->
<a href="/menu/single-origin">Click here</a>
How to Verify
  • 1Check keywords you already rank for via Search Console → Performance → find queries at position 11-20 worth optimizing
  • 2Search your target keyword on Google directly, see what kind of content ranks 1-5, build something comparable
  • 3Use Hemingway Editor to check readability and flag overly long sentences
  • 4Track results in Search Console over a 4-8 week window after publishing — SEO is a long-term process

Off-Page SEO — What Happens Outside Your Website

Signals you can't directly control, yet they account for more than 50% of total ranking factor weight

12.1 Backlinks — Why They Still Matter in 2025-2026

💡 Google emphasizes that quality matters far more than quantity — one backlink from a credible site is worth more than hundreds from low-quality directories. Buying/selling links directly violates Google's guidelines, risking a manual action or de-indexing.
  • →Digital PR — create content with original data/statistics and pitch it to media for citation
  • →Guest posting — write for other sites in your field in exchange for a backlink
  • →Resource page links — "best tools for..." list sites often welcome submissions
  • →Anchor text diversity — mix branded, exact match, natural language; don't repeat the same phrase
Anchor Text — Right vs WrongHTML
<!-- ✅ Descriptive and varied -->
<a href="https://aunjai-cafe.com">AunJai Café</a>

<!-- ❌ Same anchor everywhere, can look manipulative -->
<a href="https://aunjai-cafe.com">Click here</a>

12.2 Brand Mentions & Social Signals

While social shares aren't a direct ranking factor, heavily shared content has a higher chance of being cited as a backlink, and frequent mentions (e.g. on Wikipedia or major news outlets) are a trust signal Quality Raters factor into E-E-A-T

How to Verify
  • 1Use Google Search Console → Links to see backlinks Google has already recorded (free)
  • 2Use Ahrefs Backlink Checker (partial free access) for a rough view of your backlink profile
  • 3Audit backlinks periodically — if you find a lot of spam links, use Search Console's Disavow Tool
  • 4Track unlinked brand mentions via Google Alerts and reach out to request a link

Local SEO — Critical for Businesses with a Physical Location

46% of Google searches have local intent · 76% of "near me" searchers visit a business within 24 hours

13.1 Google Business Profile (GBP)

💡 GBP signals account for the largest share of Local Pack ranking weight (≈32% per Whitespark/BrightLocal) — 100% complete profiles get 7x more clicks than incomplete ones
  • ✓Choose the most precise primary category (the single highest-impact factor)
  • ✓Add relevant secondary categories (4+ categories = better map ranking)
  • ✓Fill in name, address, phone, hours 100% completely
  • ✓Add real, regularly updated photos (more photos = +70% location visits)
⚠️ Never stuff keywords into the business name beyond legal name/signage — Google's August 2025 Spam Update directly penalizes this ("Business Name Stuffing")

13.2 Reviews

Reviews account for ≈16% of local ranking weight, and 87-98% of consumers read reviews before deciding — ask real customers for reviews, respond to every review professionally, never buy fake reviews

13.3 NAP Consistency

NAP = Name, Address, Phone must match across every platform — 64% of small businesses have at least one NAP inconsistency, which confuses Google and AI systems and reduces trust

How to Verify
  • 1Search for your business using real customer terms on mobile, check if you appear in the Local Pack
  • 2Use Google Business Profile Performance to view monthly searches/clicks/direction requests
  • 3Check NAP consistency by searching name+phone and comparing data across sources
  • 4Check for duplicate GBP listings

Competitive Analysis & Measurement — Closing the SEO Loop

The 4 pillars covered so far mean nothing without knowing what competitors do and whether your efforts work

14.1 Content Gap Analysis

Simple Gap Analysis ProcessProcess
1. Identify your SEO competitors (may differ from business competitors)
2. Compare keywords competitors rank for that you don't
3. Compare content depth
4. Compare backlink profiles
5. Prioritize: high opportunity + achievable first
⚠️ Keyword gap analysis alone isn't enough — sometimes the real gap is a missing "journey stage," like having only sales content with nothing for the comparison stage

14.2 SEO KPIs Worth Tracking

LevelKPIWhere to Measure
VisibilityImpressions, average positionGoogle Search Console
CaptureClicks, CTRGoogle Search Console
EngagementTime on page, bounce rateGoogle Analytics 4
OutcomeConversions, leads, salesGA4 + CRM

14.3 What Timeline to Expect

SEO TimelineOutline
Months 1-3:   Technical fixes deployed, content publishing begins
Months 3-6:   Early movement on lower-competition keywords
Months 6-12:  Clearer results overall, E-E-A-T and backlinks accumulate
12+ months:  Compounding returns — old content keeps working
How to Verify
  • 1Set up Google Analytics 4 connected to Search Console for visibility + conversion data in one place
  • 2Build a list of 3-5 real SEO competitors and track their rankings monthly
  • 3Review results every 4-8 weeks, not daily — SEO has normal short-term volatility
  • 4Re-run a technical audit every quarter
